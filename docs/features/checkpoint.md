# 检查点（Checkpoint）

检查点让你随时保存进度，遇到问题可随时回退。

## 保存检查点

```bash
omc checkpoint save "完成用户模块核心逻辑"
```

## 列出检查点

```bash
omc checkpoint list
```

输出示例：

```
ID        时间                  消息                        Agent 数
---------|---------------------|---------------------------|--------
chk_001  2026-04-10 14:32    完成用户模块核心逻辑          5
chk_002  2026-04-10 13:55    完成订单模块设计              3
```

## 恢复到检查点

```bash
omc checkpoint restore chk_001
```

## 工作原理

```
每次 checkpoint save
      │
      ▼
  保存当前工作目录快照
  保存 Agent 状态
  保存已完成步骤记录
      │
      ▼
  restore 时恢复全部状态
```

## 与 Git 的区别

| | Git | Checkpoint |
|---|---|---|
| 保存内容 | 代码文件 | 代码 + Agent 状态 + 步骤记录 |
| 回退粒度 | 文件/行 | 整个任务上下文 |
| 用途 | 版本管理 | 任务进度管理 |
