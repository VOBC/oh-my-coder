## 🛡️ 数据安全

### 本地运行

Oh My Coder **完全在本地运行**：

- ✅ 你的代码不会上传到我们的服务器
- ✅ 所有处理都在你的机器上进行
- ✅ 只有你选择发送给 AI 模型的内容才会离开本地

### 数据流

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  本地代码    │ ──▶ │  Oh My Coder │ ──▶ │  AI 模型API │
└─────────────┘     │  (本地处理)  │     │ (你的密钥)  │
                    └─────────────┘     └─────────────┘
                         │
                         ▼
                    ┌─────────────┐
                    │  本地输出    │
                    └─────────────┘
```

### 我们不收集

- ❌ 你的代码内容
- ❌ 你的 API 密钥
- ❌ 你的项目信息
- ❌ 任何个人数据

## 🔑 API 密钥安全

### 密钥存储

API 密钥存储在本地环境变量中：

```bash
# 方式 1: 环境变量（推荐）
export DEEPSEEK_API_KEY=your_key_here

# 方式 2: .env 文件（开发环境）
cp .env.example .env
# 编辑 .env 填入密钥
```

### 密钥保护

- ✅ **永远不要**将密钥提交到 Git
- ✅ **永远不要**在代码中硬编码密钥
- ✅ **永远不要**在公开渠道分享密钥
- ✅ `.env` 文件已添加到 `.gitignore`

### 密钥泄露处理

如果你不小心泄露了密钥：

1. **立即**登录模型提供商控制台
2. 撤销/重置该密钥
3. 生成新密钥
4. 更新本地配置

## 🧪 代码安全

### 安全审查

生成的代码会经过以下安全检查：

1. **SecurityReviewerAgent** 自动扫描：
   - SQL 注入风险
   - XSS 漏洞
   - 敏感信息泄露
   - 不安全的依赖

2. **GitMasterAgent** 提供：
   - 修改预览（Diff）
   - 版本控制
   - 回滚能力

### 执行安全

- 所有代码在你的本地环境执行
- 没有"一键执行"危险操作
- 敏感操作需要你确认

### 依赖安全

- 我们定期检查依赖漏洞
- 使用 `pip-audit` 扫描已知漏洞
- 保持依赖更新

## ✅ 最佳实践

### 1. 环境隔离

```bash
# 使用虚拟环境
python -m venv venv
source venv/bin/activate
```

### 2. 密钥管理

```bash
# 使用环境变量，不要硬编码
export DEEPSEEK_API_KEY=$(cat ~/.secrets/deepseek_key)
```

### 3. 代码审查

- 始终审查 AI 生成的代码
- 运行测试确保正确性
- 使用 CodeReviewerAgent 审查

### 4. 版本控制

```bash
# 提交前查看变更
git diff

# 使用 GitMasterAgent 审查
python -m src.cli run "审查最近的代码变更" -w review
```

### 5. 最小权限

- 只授予必要的 API 权限
- 定期轮换 API 密钥
- 监控 API 使用量

## 🐛 漏洞报告

如果你发现安全漏洞，请负责任地披露：

### 报告方式

- **邮箱**: 请在 GitHub Issues 中创建安全相关 issue
- **响应时间**: 我们会在 48 小时内回复

### 报告内容

请包含：

1. 漏洞描述
2. 复现步骤
3. 影响范围
4. 可能的修复方案（如有）

### 我们承诺

- 及时回复
- 快速修复
- 在修复后公开致谢
