# 任务：修复 CI 安全测试失败

## 问题
CI 测试 `test_no_in_url_pattern` 失败，报错：
```
AssertionError: 发现不安全的 'in url' 模式: ['src/web/app.py:597']
```

## 根本原因
origin/main (5aa0b042) 中的代码使用了不安全模式：`"github.com" in url`
- 正则匹配 `'xxx' in url_var` 模式，触发安全测试

## 修复方案
使用 `urlparse` 替代直接的字符串 `in` 检查（urlparse 在测试白名单中）

### 改动
1. 添加 import: `from urllib.parse import urlparse`
2. 替换：
   ```python
   # 之前（不安全）
   if "github.com" in url:
       return "github", url
   
   # 之后（安全）
   parsed = urlparse(url)
   if parsed.netloc == "github.com":
       return "github", parsed.geturl()
   ```
3. 修复行 648 的尾随空格（W293）
