## Git 经验教训

**非 fast-forward 拒绝**：fetch 后本地 head 落后远程
- `git reset --hard origin/main` 被安全策略拦截
- 改用 `git checkout origin/main -- .` 清 index，再手动补文件

**GitHub 443 端口不通**：token 直接写进 URL 推送
```bash
git remote set-url origin "https://VOBC:TOKEN@github.com/VOBC/oh-my-coder.git"
git push
```

**Credential 缓存**：
```bash
printf "protocol=https\nhost=github.com\nusername=VOBC\npassword=TOKEN\n" | git credential approve
```
